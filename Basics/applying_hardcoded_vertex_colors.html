1.3 Applying Hardcoded Vertex Colors

Welcome to the fourth tutorial! In this lesson, we'll once again focus on drawing a triangle. I understand that it might seem a bit repetitive, and our progress may appear slow as we're still dealing with triangles. However, bear with me because our approach is deliberate. We aim to take small, incremental steps by making only a few code updates at a time. Building a solid understanding of the GPU programming model is essential before delving into more complex examples.

Rest assured, our pace will accelerate once we reach the second chapter. So, in this updated tutorial, we'll make slight changes to the code. Specifically, we'll continue to use the same rendering pipeline, but this time, we'll modify the shader.

Previously, we hard-coded the output color in the fragment shader. In this tutorial, we'll still hard-code the output color but in the vertex stage rather than the fragment stage. We'll pass this hard-coded color from the vertex stage to the fragment stage. The goal is to illustrate how values are passed between different stages of the pipeline.

First, as you can see, I've modified the vertex output struct by introducing a new field called color. Since there are no built-ins for a vertex color, we're using location 0 to store it. At the end of the vertex stage, we assign a hard-coded color to out.color.

In the fragment stage, we can access this color from the input and directly pass it as the output. This is a straightforward change, and you'll observe the same triangle being rendered.

Now, I have a question for our readers. In the vertex stage, we process individual vertices, while in the fragment stage, we deal with individual fragments. Remember, a fragment is conceptually similar to a pixel but can contain rich metadata such as depth and other values.

Between the vertex and fragment stages, there's an automatic conversion from geometry data to fragments handled by the GPU.

Here's the interesting part: If we assign a different color to each vertex, how will the GPU assign colors to the fragments, especially for those fragments that lie in the middle of the triangle and not directly on any of the vertices?

<script id="shader" type="wgsl">
// Vertex shader

struct VertexOutput {
    @builtin(position) clip_position: vec4<f32>,
    @location(0) color: vec3<f32>,
};

@vertex
fn vs_main(
    @location(0) inPos: vec3<f32>,
) -> VertexOutput {
    var out: VertexOutput;
    out.clip_position = vec4<f32>(inPos, 1.0);
    out.color = vec3<f32>(0.0, 0.0, 1.0);
    return out;
}

// Fragment shader

@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    return vec4<f32>(in.color, 1.0);
}

</script>
