2.3 Video Rendering

We have seen how to load and render images as textures. It is also possible to load a video as a texture. This is very useful if we want to create a web based video processing application.

a video can be viewed as nothing more than a set of images. so it is not too much different from image textures. The question is how to update the images. In this tutorial, we will look at two methods. One method requires manual update whenever a new frame is loaded. The other approach does the image update automatically via external texture.

Manual Frame Update

First, we need to introduce a function, given a url to a video file, we install a hidden video tag in the document. and we monitor the moment the video plays.

    let copyVideo = false;
    let videoTag = null;

    function setupVideo(url) {
        const video = document.createElement('video');

        let playing = false;
        let timeupdate = false;

        video.playsInline = true;
        video.muted = true;
        video.loop = true;

        video.addEventListener('playing', () => {
            playing = true;
            checkReady();
        }, true);

        video.addEventListener('timeupdate', () => {
            timeupdate = true;
            checkReady();
        }, true);

        video.src = url;
        video.play();

        function checkReady() {
            if (playing && timeupdate) {
                copyVideo = true;
            }
        }

        return video;
    }

we need to watch both playing and timeupdate events. Only both have happened, we can be sure that the video has started. if we don't wait for it being started, not only there is no video data available, even the video size could be unassigned, hence we need to wait for video start. then we need to create a texture map

        const textureDescriptor = {
            size: { width: videoTag.videoWidth, height: videoTag.videoHeight },
            format: 'rgba8unorm',
            usage: GPUTextureUsage.TEXTURE_BINDING | GPUTextureUsage.COPY_DST | GPUTextureUsage.RENDER_ATTACHMENT
        };
        const texture = device.createTexture(textureDescriptor);

        videoTag.ontimeupdate = async (event) => {
            let imageData = await createImageBitmap(videoTag);
            device.queue.copyExternalImageToTexture({ source: imageData }, { texture }, textureDescriptor.size);
        };

the size of the texture map is the same as that of the video tag. notice that we need to request RENDER_ATTACHMENT usage for this texture map, because this is required by copyExternalImageToTexture function call. finally, we create another event listener for the timeupdate event. in this event handler, we create a bitmap using the createImageBitmap function. And we load its content to the texture map via copyExternalImageToTexture.

the rest of the rendering code is the same as before, we will skip the details.

Importing External Textures

in the previous example, we manually perform a data copy whenever there is a video frame update. Another approach is importing an external texture from the video tag. This way, we avoid creating a texture map ourselves. But we need to be careful about the lifetime of the imported texture map. it can become expired if the video has switched to a new frame, or when the VideoFrame's close() function is called.

function render() {
            requestAnimationFrame(render);

            if (copyVideo) {

                const externalTexture = device.importExternalTexture({ source: videoTag });

                ...

                let uniformBindGroup = device.createBindGroup({
                    layout: uniformBindGroupLayout,
                    entries: [
                        {
                            binding: 0,
                            resource: {
                                buffer: translateMatrixUniformBuffer
                            }
                        },
                        {
                            binding: 1,
                            resource: {
                                buffer: projectionMatrixUniformBuffer
                            }
                        },
                        {
                            binding: 2,
                            resource: externalTexture
                        },
                        {
                            binding: 3,
                            resource:
                                sampler
                        }
                    ]
                });

                commandEncoder = device.createCommandEncoder();

                passEncoder = commandEncoder.beginRenderPass(renderPassDesc);
                passEncoder.setViewport(0, 0, canvas.width, canvas.height, 0, 1);
                passEncoder.setPipeline(pipeline);
                passEncoder.setBindGroup(0, uniformBindGroup);
                passEncoder.setVertexBuffer(0, positionBuffer);
                passEncoder.setVertexBuffer(1, texCoordsBuffer);
                passEncoder.draw(4, 1);
                passEncoder.end();

                device.queue.submit([commandEncoder.finish()]);
            }
        }

to void texture expiration, it's important that we import and use the texture immediately in the same render function call. and is also the reason that we create a bindgroup for each rendering frame to incorporated the new external texture map.
