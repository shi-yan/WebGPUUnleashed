2.0 Rendering to Textures

In this section, we'll explore the "render to texture" technique, a concept you might find familiar from our previous discussion on mipmaps. The "render to texture" technique involves two rendering passes. During the first pass, we use a texture map as the render target to draw something. In the second pass, we render again using that texture map. This technique is immensely useful, for instance, in creating realistic mirrors or dynamic reflections in a 3D scene.

Let's dive into the code. In this example, we'll work with two shaders: the teapot_shader and the box_shader. Our goal is to render a rotating teapot in the first pass onto a texture map and then apply that texture to a cube during the second rendering pass.

We'll encapsulate these objects in two separate classes, Teapot and Box. You've encountered the two classes before in previous examples. For simplicity, we'll render only the teapot's normals instead of applying full Phong shading.

<script>
    class Box {
        ...
    }

    class Teapot {
        ...
    }

    async function webgpu() {
        ...
        const teapotTextureSize = 512;

        const depthTextureForTeapotDesc = {
            size: [teapotTextureSize, teapotTextureSize, 1],
            dimension: '2d',
            format: 'depth24plus-stencil8',
            usage: GPUTextureUsage.RENDER_ATTACHMENT
        };

        let depthTextureForTeapot = device.createTexture(depthTextureForTeapotDesc);
        let depthTextureViewForTeapot = depthTextureForTeapot.createView();

        const depthAttachmentForTeapot = {
            view: depthTextureViewForTeapot,
            depthClearValue: 1,
            depthLoadOp: 'clear',
            depthStoreOp: 'store',
            stencilClearValue: 0,
            stencilLoadOp: 'clear',
            stencilStoreOp: 'store'
        };
        const colorTextureForTeapotDesc = {
            size: [teapotTextureSize, teapotTextureSize, 1],
            dimension: '2d',
            format: 'bgra8unorm',
            usage: GPUTextureUsage.RENDER_ATTACHMENT | GPUTextureUsage.TEXTURE_BINDING 
        };
        let colorTextureForTeapot = device.createTexture(colorTextureForTeapotDesc);

        let colorTextureForTeapotView = colorTextureForTeapot.createView();
        let colorForTeapotAttachment = {
            view: colorTextureForTeapotView,
            clearValue: { r: 0, g: 1, b: 0, a: 1 },
            loadOp: 'clear',
            storeOp: 'store'
        };

        const renderPassForTeapotDesc = {
            colorAttachments: [colorForTeapotAttachment],
            depthStencilAttachment: depthAttachmentForTeapot
        };

        const depthTextureDesc = {
            size: [canvas.width, canvas.height, 1],
            dimension: '2d',
            format: 'depth24plus-stencil8',
            usage: GPUTextureUsage.RENDER_ATTACHMENT
        };

        let depthTexture = device.createTexture(depthTextureDesc);
        let depthTextureView = depthTexture.createView();

        const depthAttachment = {
            view: depthTextureView,
            depthClearValue: 1,
            depthLoadOp: 'clear',
            depthStoreOp: 'store',
            stencilClearValue: 0,
            stencilLoadOp: 'clear',
            stencilStoreOp: 'store'
        };

        async function render() {
            ...

            let colorTexture = context.getCurrentTexture();
            let colorTextureView = colorTexture.createView();

            let colorAttachment = {
                view: colorTextureView,
                clearValue: { r: 1, g: 0, b: 0, a: 1 },
                loadOp: 'clear',
                storeOp: 'store'
            };

            const renderPassDesc = {
                colorAttachments: [colorAttachment],
                depthStencilAttachment: depthAttachment
            };


            commandEncoder = device.createCommandEncoder();
            ...

            passEncoder = commandEncoder.beginRenderPass(renderPassForTeapotDesc);
            passEncoder.setViewport(0, 0, teapotTextureSize, teapotTextureSize, 0, 1);
            teapot.encode(passEncoder);
            passEncoder.end();

            passEncoder = commandEncoder.beginRenderPass(renderPassDesc);
            passEncoder.setViewport(0, 0, canvas.width, canvas.height, 0, 1);
            box.encode(passEncoder);
            passEncoder.end();
            ...
        }
        ...
    }
</script>

To set up the pipeline, we begin by creating a teapot texture with a size of 512x512 and a depth map to handle occlusion. For rendering purposes, this target texture map must support both RENDER_ATTACHMENT and TEXTURE_BINDING.

The rotation of the teapot occurs during rendering, and after completing the teapot pass, we render the cube with the teapot's texture applied to its surface. The result is a cube with the teapot's rotation visibly reflected on its surface.
